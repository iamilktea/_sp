
# P0 編譯器 `while` 語法擴充與函數呼叫機制

## 一、 為 P0 編譯器加上 `while` 語法

在編譯器中，`while` 迴圈屬於**控制流敘述（Control Flow Statement）**。其實作核心在於利用條件跳轉與無條件跳轉指令，搭配標籤（Label）來實現反覆執行的邏輯。

### 1. 設計原理與控制流

編譯器在處理 `while condition do statement` 時，會在底層生成兩個虛擬標籤來控制執行流程：

* **`L_start`（迴圈起點）**：每次迴圈本體執行完後，或是初次進入迴圈時，用來重新評估條件的起點。
* **`L_end`（迴圈終點）**：當條件判斷為假（false）時，程式碼跳出迴圈的降落點。

**底層控制流邏輯：**

1. 標示起點 `L_start`。
2. 計算條件判斷式，若結果為假，則條件跳轉至 `L_end`。
3. 執行迴圈本體敘述。
4. 無條件跳轉回 `L_start`。
5. 標示終點 `L_end`。

### 2. 語法分析器（Parser）實作程式碼

在遞迴下降分析器中，處理敘述的函式需要加入對 `while` 關鍵字的解析。以下為實作的虛擬程式碼：

```python
def whileStatement():
    # 產生兩個唯一的標籤編號
    loop_start = create_label()
    loop_end = create_label()
    
    # 輸出「迴圈起點標籤」
    emit_label(loop_start)
    
    # 消耗 'while' 關鍵字
    match(WHILE)
    
    # 解析條件運算式 (結果會留在堆疊頂端)
    condition_type = expression() 
    if condition_type != boolean_type:
        error("While 迴圈的條件必須是布林型態")
        
    match(DO)
    
    # 輸出「條件為假時跳轉」的指令 (跳到 loop_end)
    emit_jump_if_false(loop_end)
    
    # 解析迴圈本體的敘述
    statement()
    
    # 執行完本體，無條件跳回起點
    emit_jump(loop_start)
    
    # 輸出「迴圈終點標籤」
    emit_label(loop_end)

```

## 二、 P0 編譯器的函數呼叫機制

P0 編譯器（及多數 C-like 語言）採用高度結構化的活動紀錄（Activation Record，又稱 Stack Frame / 堆疊框）**機制。系統透過連續的**呼叫堆疊（Call Stack）來管理函數切換時的狀態保存與變數存取。

函數呼叫的生命週期由主調端（Caller）與被調端（Callee）共同配合，分為以下三個核心階段：

### 1. 呼叫前準備（Caller 職責）

當主程式準備呼叫函數時，會先佈置好進入前的堆疊狀態：

* **傳遞參數**：將目標函數需要的參數依序推入（Push）堆疊。
* **推入返回位址**：執行 `CALL` 指令，硬體會自動將「下一條指令的記憶體位址」推入堆疊，確保函數執行完畢後知道該回到哪裡。

### 2. 進入函數與初始化（Callee Prologue - 序言）

目標函數剛開始執行時，必須先建立自己的堆疊框，確保能獨立運作且不破壞原程式狀態：

* **保存舊基址（Push BP）**：將目前的基址指標暫存器（BP）推入堆疊保存。
* **更新基址（MOV BP, SP）**：將目前的堆疊指標（SP）複製給 BP。此時 BP 成為該函數專屬的定址基準點。
* **建立靜態鏈（Static Link）**：若支援巢狀函數，需推入指向外層函數 BP 的指標，以便跨層存取外層變數。
* **配置區域變數**：將 SP 往下移動（`SUB SP, space_size`），預留記憶體空間給內部的區域變數使用。

**變數定址原則：**
編譯器利用 BP 的相對偏移量來存取資料。`BP + 偏移量` 用來讀取上方的參數與返回位址；`BP - 偏移量` 用來讀取下方的區域變數。

### 3. 離開函數與清理（Callee Epilogue - 結語）

當函數準備 `return` 時，必須清理自己建立的堆疊並交還控制權：

* **設定回傳值**：將執行結果放入預設暫存器（如 `AX`）。
* **釋放空間（MOV SP, BP）**：將 SP 移回 BP 的位置，瞬間回收所有區域變數與靜態鏈的空間。
* **還原舊基址（Pop BP）**：從堆疊頂端彈出舊的 BP 值，恢復主調端的堆疊狀態。
* **返回原程式（RET）**：執行返回指令，彈出返回位址並載入程式計數器（PC），程式完美回到呼叫處繼續執行。
